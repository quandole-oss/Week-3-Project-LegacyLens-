## Option C: LinkedIn

**Title suggestion:** Built a RAG App to Understand 200K Lines of Legacy Fortran

**Post:**

Understanding legacy code is one of the hardest problems in software engineering. What if AI could help?

I built **LegacyLens** during Week 3 at Gauntlet AI — a RAG-powered application that makes LAPACK (200K+ lines of numerical Fortran from the 1990s) searchable and explainable in plain English.

**What it does:**
- Ask natural language questions about the codebase
- Semantic search across 2,306 source files (4,620 vector embeddings)
- Auto-generate modern documentation from legacy headers
- Map subroutine dependencies and call chains
- Detect coding patterns across the codebase
- Extract core algorithms from dense Fortran routines

**Tech stack:** FastAPI, React/TypeScript, Pinecone, LangChain, Claude, OpenAI embeddings

**By the numbers:**
- 2,306 Fortran source files ingested
- 4,620 vector embeddings in Pinecone
- 7 distinct code understanding features
- 235 automated tests (pytest + vitest)
- Deployed on Railway + Vercel

Try it live: https://legacylens-murex.vercel.app
